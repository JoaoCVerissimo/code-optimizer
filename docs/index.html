The Workflow

How It Works

From pasting your code to seeing benchmark results, everything happens in six clear steps.

1

You Submit Your Code

Paste a function or code snippet into the editor. Pick the language you're working in and choose what you'd like to optimize for.

The editor supports Python, JavaScript, TypeScript, and Go with full syntax highlighting.
2

AI Analyzes and Optimizes

Your code is sent to Claude AI along with a specialized prompt for your chosen goal. The AI studies your code, identifies improvement opportunities, and generates three different optimized versions — each using a unique strategy.

For example, a slow recursive function might get transformed into a memoized version, an iterative version, and a mathematically optimized version.
3

Sandboxed Benchmarking

Each version of your code — including the original — is executed inside an isolated Docker container. Every variant runs three times to get reliable measurements. We capture execution time, CPU usage, and memory consumption.

Sandboxes are completely isolated: no network access, strict resource limits, and they're destroyed after each run. Your code can't affect anything outside its container.
4

Scoring and Ranking

Benchmark results are normalized into scores from 0 to 100 across five dimensions: performance, memory, security, reliability, and readability. The scores are weighted based on your chosen optimization goal, and every variant gets ranked.

If you chose "performance", execution speed counts for 60% of the overall score. If you chose "reliability", fault tolerance gets the highest weight instead.
5

Visual Comparison

Results appear as interactive charts so you can instantly see how each variant compares. Bar charts show raw execution time and memory, while radar charts overlay all score dimensions for a complete picture.

You can also view each variant's code side-by-side with the original, along with the AI's explanation of what it changed and why.
6

Pick the Winner

The system highlights the best-performing variant, but you have all the data to make your own choice. Maybe the fastest version sacrifices readability — the scores help you find the right trade-off.

Every submission is saved in your history so you can come back and compare past optimizations anytime.

Safety First

Sandbox Security

Every piece of code runs inside a locked-down, disposable container. Here's what keeps things safe.

🔒

No Network Access

Containers cannot make any network connections

📈

256 MB Memory Cap

Hard limit prevents runaway memory usage

⚡

1 CPU Core Limit

Prevents CPU starvation of the host

🛡

64 Process Limit

Fork bombs are blocked at 64 processes

👤

Non-Root User

Code runs as unprivileged user (uid 1000)

⏰

30-Second Timeout

Hard kill if execution exceeds time limit

🚫

All Capabilities Dropped

Minimal kernel interface, no privilege escalation

🗑

Ephemeral Containers

Created, used once, then destroyed completely

Get Started

Run It Locally

Everything runs on your machine using Docker. No paid services, no cloud dependencies.

Quick Start (Docker)

# Clone and configure
git clone <repo-url> code-optimizer
cd code-optimizer
cp .env.example .env
# Edit .env and set your ANTHROPIC_API_KEY

# Build sandbox images (one-time)
make sandbox-images

# Start everything
make docker-up

# Open http://localhost:3000

Development Setup (with hot reload)

# Install dependencies
npm install

# Start infrastructure (Postgres + Redis)
docker compose up -d postgres redis

# Run database migrations
DATABASE_URL=postgres://optimizer:optimizer_dev@localhost:5432/code_optimizer make migrate

# Start all apps in dev mode
make dev

# Frontend: http://localhost:3000
# API:      http://localhost:4000
